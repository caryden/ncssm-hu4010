### DMU Section (Slides 7-17)

#### Slide 7: Section Divider
- **Type:** `slide-statement`
- **Content:**
  - Statement: "Step 12: Determine the Decision-Making Unit"
- **Visual:** DE wheel with Step 12 glowing

#### Slide 8: Who Decides?
- **Type:** `slide-visual`
- **Content:**
  - Title: "Decisions Involve Multiple People"
- **Visual:** Three examples:
  - Teen buying phone → Parents have veto
  - Employee choosing software → IT and manager
  - Spouse buying furniture → Partner weighs in
- **Animation:** Each example appears with icons showing the stakeholders
- **Point:** "Even 'simple' decisions involve others"

#### Slide 9: The DMU Framework
- **Type:** `slide-visual`
- **Content:**
  - Title: "Six DMU Roles"
- **Visual:** Circular arrangement of 6 roles around a central "Purchase Decision":
  1. Champion (star icon)
  2. End User (user icon)
  3. Economic Buyer (dollar icon)
  4. Technical Buyer (settings icon)
  5. Influencer (megaphone icon)
  6. Veto Power (x-circle icon)
- **Animation:** Each role appears with icon and brief label
- **Color:** Each role has subtle color variation

#### Slide 10: Role - Champion
- **Type:** `slide-two-part`
- **Content:**
  - Primary: "Champion"
  - Secondary: "Your internal advocate who believes in your solution"
  - Cares about:
    - Looking good by bringing a winning solution
    - Having tools to sell internally
    - Success metrics to share
- **Visual:** Star icon, enthusiastic person
- **Animation:** Role details appear

#### Slide 11: Role - End User
- **Type:** `slide-two-part`
- **Content:**
  - Primary: "End User"
  - Secondary: "The person who actually uses the product day-to-day"
  - Cares about:
    - Ease of use
    - Time savings
    - Making their job easier
- **Note:** "May not have buying power but can block adoption"
- **Visual:** User icon with tools

#### Slide 12: Role - Economic Buyer
- **Type:** `slide-two-part`
- **Content:**
  - Primary: "Economic Buyer"
  - Secondary: "Controls the budget and makes the financial decision"
  - Cares about:
    - ROI and cost
    - Risk mitigation
    - Strategic alignment
- **Note:** "Often not the person you first talk to"
- **Visual:** Dollar icon, executive figure

#### Slide 13: Role - Technical Buyer
- **Type:** `slide-two-part`
- **Content:**
  - Primary: "Technical Buyer"
  - Secondary: "Evaluates fit with existing systems"
  - Cares about:
    - Security and compliance
    - Integration complexity
    - Maintenance burden
- **Note:** "Can veto for technical reasons"
- **Visual:** Settings/gear icon, IT person

#### Slide 14: Roles - Influencer & Veto
- **Type:** `slide-comparison`
- **Content:**
  - Left (Influencer):
    - Shapes opinions but doesn't decide
    - Could be: analysts, consultants, peers
    - Important for credibility
  - Right (Veto Power):
    - Can say NO but not YES
    - Often: Legal, Compliance, Security
    - Must be addressed early
- **Animation:** Side-by-side reveal
- **Visual:** Megaphone vs X-circle icons

#### Slide 15: Different Motivations
- **Type:** `slide-visual`
- **Content:**
  - Title: "Each Role Has Different Concerns"
- **Visual:** Table/matrix showing:
  - End User: "Will this make my job easier?"
  - Economic Buyer: "Will this save/make us money?"
  - Technical Buyer: "Will this integrate safely?"
  - Legal/Compliance: "Does this meet our requirements?"
- **Animation:** Each row appears with the question
- **Key insight:** "You need to address ALL of these"

#### Slide 16: DMU Map Example
- **Type:** `slide-visual`
- **Content:**
  - Title: "Example: Selling Project Management Software"
- **Visual:** DMU map showing:
  - Center: Purchase decision
  - Champion: Project Manager (found the tool)
  - End Users: Team members (will use daily)
  - Economic Buyer: VP of Operations (budget)
  - Technical Buyer: IT Director (security)
  - Influencer: Industry consultant (recommended it)
  - Veto: Security team (data concerns)
- **Animation:** Map builds with relationships drawn
- **Lines:** Show influence relationships

#### Slide 17: Exercise Setup
- **Type:** `slide-statement`
- **Content:**
  - Statement: "Map YOUR Decision-Making Unit"
- **Subtext:** "15 minutes: Identify stakeholders, roles, and what each cares about"
- **Animation:** Transition to workshop

#### Slide 24: Example Acquisition Map
- **Type:** `slide-visual`
- **Content:**

  - Interest → Evaluation (Free trial signup, 1-7 days)
  - Evaluation → Decision (Sales call, 1-4 weeks)
  - Decision → Purchase (Contract, 1-3 days)
  - Purchase → Onboarding (Setup, 1-4 weeks)
- **Animation:** Steps connect with timing labels
- **Total:** "6-10 weeks from first touch to active user"

#### Slide 25: B2B vs B2C
- **Type:** `slide-comparison`
- **Content:**
  - Left (B2C):
    - Decision time: Minutes to days
    - Stakeholders: 1-3 people
    - Sale type: Self-service
    - Risk tolerance: Higher
  - Right (B2B):
    - Decision time: Weeks to months
    - Stakeholders: 5-15 people
    - Sale type: Sales-assisted
    - Risk tolerance: Lower
- **Animation:** Side-by-side comparison builds
- **Point:** "Know which world you're in"

## Key Visualizations to Build

### 1. DMU Circle Diagram (Slide 9)
```
Structure:
- Central node: "Purchase Decision"
- 6 surrounding nodes for each role
- Icons for each role
- Connecting lines showing influence

Animation:
1. Central node appears
2. Each role node appears with icon
3. Role labels fade in
4. Connection lines draw

Technical:
- D3 force layout (fixed positions)
- SVG icons from Lucide
- Subtle animation on connections
```

### 2. Acquisition Funnel (Slide 19)
```
Structure:
- Tapered funnel shape
- 6 horizontal bands (stages)
- Labels for each stage
- Arrows showing flow

Animation:
1. Funnel outline draws
2. Top band fills (Awareness)
3. Each subsequent band fills
4. Flow arrows animate between stages

Technical:
- SVG path for funnel shape
- Gradient fill (light → dark green)
- Staggered animation timing
```

### 3. Process Timeline with DMU (Slide 24)
```
Structure:
- Horizontal timeline
- Stage nodes with labels
- Duration between stages
- DMU role indicators at each stage

Animation:
1. Timeline base draws
2. First stage node appears
3. Duration label appears
4. Next stage node
5. Continue through process
6. Total time calculation at end

Technical:
- SVG line + circles for nodes
- Animated text for durations
- Counter for total time
```

### 4. DMU Relationship Map (Slide 16)
```
Structure:
- Central decision node
- Stakeholder nodes arranged around
- Relationship lines with labels
- Influence direction arrows

Animation:
1. Decision node appears center
2. Champion appears (first to find you)
3. Relationships draw outward
4. Other stakeholders appear
5. All connections complete

Technical:
- D3 force layout
- Curved connection paths
- Arrow markers for direction
- Labels on hover/click (optional)
```

### 5. B2B vs B2C Comparison (Slide 25)
```
Structure:
- Two columns
- Matching rows for comparison
- Visual indicators for differences

Animation:
1. Headers appear (B2C, B2B)
2. First comparison row
3. Subsequent rows
4. Key differences highlighted

Technical:
- CSS grid layout
- Staggered row animations
- Color coding for differences
```
