### End User Profile Section (Slides 6-12)

#### Slide 6: Section Divider
- **Type:** `slide-statement`
- **Content:**
  - Statement: "Step 3: Build an End User Profile"
- **Visual:** DE wheel with Step 3 glowing

#### Slide 7: End User vs. Economic Buyer
- **Type:** `slide-visual`
- **Content:**
  - Title: "Who Actually Uses It?"
- **Visual:** Split comparison showing:
  - Consumer app: User = Buyer (same person icon)
  - Enterprise SW: User ≠ Buyer (two different icons)
  - Kids' game: User = Child, Buyer = Parent
- **Animation:** Icons appear in pairs, relationship shown
- **Key point:** "Design for the user, sell to the buyer"

#### Slide 8: Profile Components
- **Type:** `slide-visual`
- **Content:**
  - Title: "What You Need to Know"
- **Visual:** Circular diagram with person silhouette at center, surrounded by:
  - Demographics (who they are)
  - Psychographics (what they value)
  - Behaviors (what they do)
  - Context (where/when problem occurs)
  - Current solutions (what they do today)
- **Animation:** Each segment appears around center

#### Slide 9: The Questions
- **Type:** `slide-list`
- **Content:**
  - Title: "Answer Through Research"
  - List:
    - What does a typical day look like?
    - Where does the problem occur?
    - How do they solve it today?
    - What frustrates them about current solutions?
    - What would "better" look like?
- **Animation:** Staggered appearance
- **Icon:** Question mark

#### Slide 10: Primary Market Research
- **Type:** `slide-two-part`
- **Content:**
  - Primary: "The 80/20 Rule"
  - Secondary: "Listen 80% of the time. Talk 20%."
- **Visual:** Visual balance scale showing listening outweighing talking
- **Animation:** Scale tips toward listening side

#### Slide 11: Interview Do's & Don'ts
- **Type:** `slide-comparison`
- **Content:**
  - Left (Do):
    - Ask open-ended questions
    - Focus on past behavior
    - Ask "why?" repeatedly
    - Take notes
  - Right (Don't):
    - Pitch your solution
    - Ask leading questions
    - Accept hypotheticals
    - Assume you understand
- **Animation:** Green checkmarks vs red X's appear
- **Color:** Success green / danger red

#### Slide 12: Magic Questions
- **Type:** `slide-list`
- **Content:**
  - Title: "The Interview Framework"
  - List:
    1. "Tell me about the last time you experienced [problem]..."
    2. "What did you do about it?"
    3. "How did that work out?"
    4. "What would make it better?"
    5. "How much time/money does this cost you?"
- **Animation:** Each question appears as if typing
- **Visual:** Speech bubble aesthetic

### TAM Section (Slides 13-19)

#### Slide 13: Section Divider
- **Type:** `slide-statement`
- **Content:**
  - Statement: "Step 4: Calculate TAM"
- **Visual:** DE wheel with Step 4 glowing

#### Slide 14: What is TAM?
- **Type:** `slide-two-part`
- **Content:**
  - Primary: "Total Addressable Market"
  - Secondary: "The total revenue if you owned 100% of your beachhead"
- **Visual:** Pie chart fully colored (100%)
- **Animation:** Pie fills completely

#### Slide 15: Why TAM Matters
- **Type:** `slide-list`
- **Content:**
  - Title: "TAM Tells You..."
  - List:
    - Is this market worth pursuing?
    - What's the ceiling for this opportunity?
    - Do you need to expand to adjacent markets?
    - Are your goals realistic?
- **Animation:** Staggered list
- **Icon:** Dollar sign / chart

#### Slide 16: The Bottom-Up Method
- **Type:** `slide-visual`
- **Content:**
  - Title: "How to Calculate TAM"
- **Visual:** Formula animation:
  ```
  TAM = Customers × Revenue/Customer
  ```
  - Step 1: Count potential customers
  - Step 2: Estimate annual revenue per customer
  - Step 3: Multiply
- **Animation:**
  1. Formula appears
  2. Each component highlights with example value
  3. Calculation animates to result

#### Slide 17: TAM Example
- **Type:** `slide-visual`
- **Content:**
  - Title: "Durham Restaurant Example"
- **Visual:** Animated calculation:
  - 200 restaurants meeting criteria
  - × $2,400/year (based on competitor pricing)
  - = $480,000/year TAM
- **Animation:** Numbers multiply visually
- **Question:** "Is this big enough?"

#### Slide 18: TAM Sanity Checks
- **Type:** `slide-list`
- **Content:**
  - Title: "Goldilocks TAM"
  - List:
    - Too small (<$1M): May not be worth pursuing
    - Just right ($5M-$50M): Beachhead sweet spot
    - Too big (>$100M): Are you being specific enough?
- **Animation:** Three bears/bowls visual metaphor
- **Visual:** Size indicators

#### Slide 19: The Key Question
- **Type:** `slide-statement`
- **Content:**
  - Statement: "What assumptions does your TAM depend on?"
- **Subtext:** "Write them down. Test them."
- **Animation:** Statement appears, then subtext fades in

## Key Visualizations to Build

### 1. Progressive Focus Animation (Slide 5)
```
Structure:
- Three stages: Crowd → Group → Individual
- Blur/sharpness metaphor
- Labels: Segment → Profile → Persona

Animation:
1. Blurry crowd of icons appears
2. Crowd sharpens to smaller group
3. Group focuses to single clear person
4. Labels appear at each stage

Technical:
- SVG blur filter with animated stdDeviation
- Scale + opacity transitions
- 3-second total animation
```

### 2. TAM Calculator (Slide 16-17)
```
Structure:
- Formula display: TAM = Customers × Revenue
- Input fields with example values
- Result with dollar formatting

Animation:
1. Formula appears
2. "Customers" value fills in (200)
3. "Revenue" value fills in ($2,400)
4. Multiplication visual (× sign pulses)
5. Result animates (counting up to $480,000)

Technical:
- D3 number interpolation for counting effect
- Currency formatting
- Highlight boxes around each component
```

### 3. Persona Card Builder (Slide 23-24)
```
Structure:
- Card container with sections
- Photo area (circular crop)
- Name + quote header
- Demographics sidebar
- Goals/Frustrations columns
- Current solution footer

Animation (build sequence):
1. Card outline appears
2. Photo placeholder → photo fills
3. Name types in
4. Quote appears in quotation marks
5. Demographics fade in
6. Goals populate (1, 2, 3)
7. Frustrations populate (1, 2, 3)
8. Current solution fills footer

Technical:
- CSS card component
- Typewriter effect for name
- Staggered list animations
- Subtle shadow/depth on card
```

### 4. Theme Completion Celebration (Slide 28)
```
Structure:
- DE wheel with Theme 1 section
- Steps 1-5 around that section
- Checkmarks for each step
- "Who is your customer? ✓" label

Animation:
1. Wheel appears (full, dimmed)
2. Theme 1 section highlights (blue glow)
3. Step 1 checkmark appears
4. Steps 2-5 checkmarks appear (staggered)
5. Label fades in
6. Brief celebration effect (particles/glow)

Technical:
- Reuse wheel component from Class 0
- SVG checkmark animation
- Canvas or SVG particles for celebration
```
